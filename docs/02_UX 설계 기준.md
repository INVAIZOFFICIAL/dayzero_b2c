## UX 설계 기준

DayZero B2C는 **쉬운 사용성**을 3대 차별화 원칙 중 하나로 선정. 

이를 실현하기 위해, **구체적이고 측정 가능한 UX 설계 기준**을 정의함.

</aside>

## 🔎 배경

<aside>
1️⃣

### **경쟁사 6개사 공통 문제 - 정보 과부하**

투플렉스, 셀러픽, 윈들리, 퍼센티, 셀잇파파, Q10 Auto 모두 소싱/가공/등록/재고 관련 데이터를 **한 화면에 과도하게 노출**. 이는 초보자가 첫 화면에서 이탈하는 핵심 원인이 될 수 있음. 

</aside>

<aside>
2️⃣

### **셋팅부터 주문까지, 단계의 복잡성**

초기 셋팅, 소싱, 가공/등록, 싱크, 주문까지 각 단계가 세부 단계를 포함. 기준 없이 설계하면 **단계마다 인지 부하가 누적**되어 ‘다 좋은데 어디서부터 해야 할지 모르겠다’는 상황이 발생할 수 있음.

</aside>

## 💡 설계 목표

<aside>
1️⃣

### **초보 셀러도 첫 상품 등록까지 막힘 없이 도달**

경쟁사 솔루션에서 초보 셀러가 이탈하는 가장 큰 이유는 ‘어디서부터 어떻게 시작해야 할지 모르겠다’는 것. DayZero는 **가입 후 첫 상품 등록 완료까지의 경로를 최소 단계로 설계**하여, 셀러가 성공 경험을 빠르게 얻도록 함.

</aside>

<aside>
2️⃣

### **반복 조작을 줄여 시간 절약**

중급 셀러의 핵심 병목은 상품 등록에 소요되는 시간. 하루 10개 등록이 목표이나 수동으로는 5개도 어렵움. DayZero는 **AI 자동 완성 + 대량 처리**로 셀러의 반복 조작 횟수를 최소화.

</aside>

## 🧑‍💻 핵심 설계 원칙

<aside>
1️⃣

### 지금 해야 할 것만 강조

선택지가 많아지면 초보자는 멈추기 때문에 한 화면에서 셀러가 해야 할 일을 명확하게 제공. 경쟁사 6개사가 공통적으로 실패한 지점은 **한 화면에 너무 많은 정보를 노출하는 것**.

- **원칙**: 화면당 핵심 액션 3개 이하. 부수적 설정은 숨기고 기본값으로 자동 처리. 작업 흐름을 단계별로 명확하게 분리.
</aside>

<aside>
2️⃣

### **시스템 선처리, 셀러 최종 확인**

셀러가 판단하기 어렵거나 반복적인 작업은 시스템이 먼저 처리. 단, 모든 필드를 무조건 채우는 것이 아니라 **필드 성격에 따라 자동화 수준을 구분**.

- **원칙**: 필드 성격별 자동화 3단계 구분 - AI 자동 완성(카테고리, 번역, 가격 환산) / 스마트 기본값 제안(마진율, 배송비) / 셀러 직접 입력(특이사항, 커스텀 설명)
</aside>

<aside>
3️⃣

### 명확한 피드백

셀러가 ‘다음에 뭐 하지?, 현재 어떤 상황이지’라고 생각하는 순간이 없어야 함. 현재 단계와 다음 행동은 항상 명확할 것. 배경에서 정의한 '할 일은 많은데 순서가 안 보인다' 문제를 직접 해결하는 원칙.

- **원칙**: 현재 위치 표시 + 다음 행동 버튼 항상 노출
</aside>

<aside>
4️⃣

### **에러 발생 시, 해결책 제시**

문제가 생겼을 때 ‘실패했습니다’로 끝나면 초보 셀러는 막힘. 에러 메시지와 함께 **구체적인 해결 방법**의 즉시 제시 필요.

- **원칙**: 모든 에러 메시지는 ‘왜 실패했는지 + 어떻게 해결하는지’ 포함
</aside>

## ✍️ UX 라이팅 원칙

<aside>
1️⃣

### 쉬운 단어 사용

크로스보더 역직구에는 전문 용어가 많지만, 초보 셀러는 HS코드, SKU, EMS 같은 단어를 모를 수 있음. 전문 용어는 반드시 괄호 설명 또는 툴팁으로 보완.

- Bad: HS코드를 입력하세요
- Good: 세관 분류코드(HS코드)를 입력하세요 - 모르면 AI가 자동 추천
</aside>

<aside>
2️⃣

### 한 문장, 한 가지 정보

DayZero는 소싱부터 주문까지 단계가 많기 때문에, 한 문장에 두 가지 이상의 행동을 요구하지 않음. 불필요한 수식어를 빼고, 필요한 정보만 전달.

- Bad: 소싱처 URL을 입력하고 카테고리를 선택한 뒤 등록 버튼을 누르세요
- Good: 소싱처 URL을 입력하세요 (다음 단계에서 카테고리 선택)
</aside>

<aside>
3️⃣

### 행동을 안내하는 문장

설명이 아니라 다음 행동을 유도. 버튼과 라벨은 클릭하면 무슨 일이 일어나는지 담아야 함.

- Bad: 상품 등록 / 번역 설정
- Good: Q10에 등록하기 / 상품명 일본어로 번역하기
</aside>

<aside>
4️⃣

### 상태를 숨기지 않기

핵심 설계 원칙 3번(명확한 피드백)의 라이팅 버전. 시스템이 뭘 하고 있는지, 얼마나 걸리는지 항상 텍스트로 표현.

- Bad: 처리 중... / 완료
- Good: AI가 카테고리를 매칭하고 있어요 (약 10초) / 3개 상품이 Q10에 등록됐어요
</aside>

<aside>
5️⃣

### 에러는 해결책과 함께

핵심 설계 원칙 4번의 라이팅 버전. 왜 + 어떻게를 한 세트로.

- Bad: 등록 실패
- Good: 이미지 크기가 초과되어 등록이 안 됐어요. 자동 리사이즈로 다시 시도할까요?
</aside>

<aside>
6️⃣

### 어조 기준 - 해요체 통일

경어체(해요체)로 통일. 딱딱한 합쇼체(-습니다)가 아니라, 도와주는 느낌의 해요체.

- 기본 어조: ~하세요, ~할 수 있어요, ~됐어요
- 지양: ~하십시오, ~바랍니다 (관공서 느낌)
</aside>

## 🔘 버튼 라벨 규칙

<aside>
➡️

### 기본 원칙 - 동사형 기본 + 예외적 명사형 혼용

DayZero의 핵심 타겟은 초보 셀러. 초보자는 이걸 누르면 뭐가 되는지를 알아야 멈추지 않음. 따라서 **핵심 행동 버튼은 동사형**, 탐색/네비게이션 영역은 명사형으로 구분.

</aside>

| **위치** | **형태** | **예시** | **이유** |
| --- | --- | --- | --- |
| Primary CTA (핵심 행동) | **동사형** | Q10에 등록하기, AI로 번역하기, 상품 소싱하기 | 클릭 결과가 명확해서 초보자도 망설이지 않음 |
| 탭 / 메뉴 라벨 | **명사형** | 소싱, 가공/등록, 주문 관리 | 짧고 깔끔. 빠른 스캔에 유리 |
| 토글 / 설정 항목 | **명사형** | 자동 번역, 가격 자동 환산 | ON/OFF 대상이므로 명사가 자연스러움 |
| 확인/완료 버튼 | **동사형** | 저장하기, 확인하기, 다음 단계로 | 행동을 유도하는 마무리 액션 |

## 💬 상황별 라이팅 패턴

<aside>
1️⃣

### 빈 상태(Empty State)

처음 가입한 셀러는 상품 목록, 주문 내역, 대시보드가 전부 비어있음. 빈 화면은 이탈의 시작점이 될 수 있으므로, **다음 행동을 유도하는 문구 + CTA 버튼**을 함께 노출.

- Bad: 등록된 상품이 없습니다
- Good: 아직 등록된 상품이 없어요. 첫 상품을 소싱해볼까요? [CTA: 상품 소싱하기]
- 주문 내역: 아직 들어온 주문이 없어요. 상품을 등록하면 여기서 주문을 확인할 수 있어요.
</aside>

<aside>
2️⃣

### 확인/경고 다이얼로그

삭제, 대량 등록, 가격 일괄 변경 등 되돌리기 어려운 행동에는 확인 단계가 필수. 확인 문구는 **무엇이 어떻게 되는지** 구체적으로 설명.

- Bad: 정말 삭제하시겠습니까?
- Good: 3개 상품이 Q10에서 내려가요. 삭제할까요?
- 대량 등록: 15개 상품을 Q10에 한꺼번에 등록해요. 진행할까요? (약 2분 소요)
- 가격 변경: 선택한 8개 상품의 마진율이 30%로 변경돼요. 적용할까요?
</aside>

<aside>
3️⃣

### 로딩/진행률 텍스트

라이팅 원칙 4번(상태를 숨기지 않기)의 상황별 확장. 처리 유형에 따라 표현 패턴을 구분.

- **단건 AI 처리**: AI가 카테고리를 매칭하고 있어요 (약 10초)
- **대량 처리**: 15개 상품 등록 중... 7/15 완료 (3번 상품에서 이미지 리사이즈 중)
- **외부 API 호출**: Q10 서버에 등록 요청을 보냈어요. 응답 대기 중... (보통 30초 이내)
- **백그라운드 작업**: 번역이 진행 중이에요. 완료되면 알려드릴게요. 다른 작업을 먼저 해도 돼요.
</aside>

<aside>
4️⃣

### 성공 상태

성공 피드백은 결과 + 다음 행동 안내를 한 세트로. 단순히 완료로 끝내지 않고, 셀러가 바로 다음 단계로 이동할 수 있게.

- Bad: 등록 완료
- Good: 3개 상품이 Q10에 등록됐어요. Q10 셀러센터에서 확인할까요? [CTA: Q10에서 확인하기]
- 번역 성공: 상품명이 일본어로 번역됐어요. 검토 후 수정할 수 있어요. [CTA: 번역 결과 확인하기]
- 대량 완료: 15개 상품 등록이 전부 완료됐어요. 2건에서 이미지가 자동 리사이즈됐어요. [CTA: 결과 상세 보기]
</aside>

## 🔄 화면 흐름 규칙

<aside>
1️⃣

### 3단계 이상이면 탭/스텝 전환

한 화면에 3단계 이상의 작업이 들어가면 탭 또는 스텝 단위로 분리. 한 화면에 모든 걸 담으려는 시도가 경쟁사 공통 실패 원인.

- **원칙**: 화면당 작업 단계는 최대 3단계. 초과 시 탭 전환 또는 스템퍼로 분리
</aside>

<aside>
2️⃣

### 부수적 설정은 숨기고 기본값 자동

핵심 설계 원칙 1번(지금 해야 할 것만 강조)의 화면 구조 버전. 고급 설정은 접힌 영역에 넣고, 가장 많이 쓰는 값을 기본값으로 세팅.

- **원칙**: 필수 입력만 노출, 선택 입력은 접기 영역 또는 고급 설정으로 분리
</aside>

<aside>
3️⃣

### 현재 단계 인디케이터 항상 표시

핵심 설계 원칙 3번(명확한 피드백)의 네비게이션 버전. 셀러가 전체 흐름에서 어디에 있는지, 다음엔 뭐가 있는지 항상 보여줌.

- **원칙**: 스템퍼, 프로그레스 바, 브레드크럼 등으로 현재 위치 명시
</aside>

<aside>
4️⃣

### 뒤로가기 시 입력값 보존

셀러가 이전 단계로 돌아갔을 때 입력한 내용이 날아가면 재입력 부담이 발생. 단계 간 이동 시 기존 입력값은 반드시 유지.

- **원칙**: 이전/다음 이동 시 모든 입력 필드 상태 보존
</aside>

<aside>
5️⃣

### 긴 작업은 백그라운드 + 진행률 위젯

대량 등록, AI 번역 등 오래 걸리는 작업은 백그라운드로 처리하고 셀러는 다른 작업을 병행할 수 있게. 진행률 위젯으로 상태 확인 가능.

- **원칙**: 10초 이상 소요 작업은 백그라운드 처리 + 진행률 표시 + 완료 알림
</aside>

## 📐 표기 규칙

<aside>
1️⃣

### 숫자 및 단위

- 가격: 통화 기호 + 콤마 표기 (예: \1,200 / $9.99)
- 퍼센트: 숫자 + % (예: 마진율 30%)
- 수량: 아라비아 숫자 사용 (예: 3개 상품, 10건 등록)
</aside>

<aside>
2️⃣

### Placeholder (입력 필드 안내)

입력 필드 안에 예시값을 넣어 셀러가 어떤 형식으로 입력해야 하는지 바로 파악할 수 있게.

- URL 입력: [https://www.oliveyoung.co.kr/](https://www.oliveyoung.co.kr/)...
- 가격 입력: 12,000
- 상품명 입력: 상품명을 입력하세요
</aside>

<aside>
3️⃣

### 일관성

위의 라이팅 원칙과 버튼 라벨 규칙을 모든 화면에서 동일하게 적용. 같은 기능은 같은 단어로, 같은 상태는 같은 표현으로 통일.

</aside>
