## Critical — 즉시 확인 필요

### C-1. "편집 완료" 버튼 미구현

**위치**: `EditingHeader.tsx`

기획서 `[ED-02]`:
> "편집 완료" 버튼 (Primary) — 현재 상품의 모든 편집 항목 확정. 종합 상태 → "편집완료" 전환

현재 코드에는 `Qoo10 등록` 버튼만 존재합니다. `editStatus` 필드가 타입에 정의되어 있지만(`types/editing.ts:47`), 사용자가 직접 확정하는 경로가 없습니다.

**영향**:
- 셀러가 "나는 이 상품 편집을 끝냈다"라고 표시할 방법이 없음
- ED-01 목록에서 "편집 완료" 탭의 의미가 기획서와 달라짐
- Qoo10 등록 활성화 조건이 기획서(`editStatus === 'completed'`)와 다름

**의사결정 필요**: UT 프로토타입에서 "편집 완료" 단계를 생략하고 번역 완료 = 편집 완료로 간주할 것인지, 기획대로 명시적 확정 버튼을 추가할 것인지.

### C-2. ED-01 탭 라벨 기획서 불일치

**위치**: `EditingListPage.tsx:21`

| 기획서 | 현재 코드 |
|--------|----------|
| 번역 필요 | 작성 필요 |
| 편집 완료 | 작성 완료 |

"작성 필요/완료"는 번역 완료 여부 기준이고, 기획서의 "편집 완료"는 `editStatus` 확정 기준입니다. C-1과 연결된 문제.

## 의사결정 기록

### "AI 번역" → "AI 편집" 네이밍 변경
- **결정**: 모든 UI 텍스트에서 "번역" → "편집"으로 통일
- **이유**: 단순 번역이 아닌 상품명 작성·옵션 편집·상세설명 생성을 포함하므로 "편집"이 더 정확
- **트레이드오프**: 기획 문서의 "AI 번역" 용어와 불일치 발생 → 기획 문서 업데이트 필요

### 비활성 버튼 UX — 하이브리드 접근
- **결정**: ED-01은 경고 + 비활성화, ED-02는 완전 비활성화 + 툴팁
- **고려한 대안**: A) 항상 비활성화, B) 항상 허용 + 경고
- **선택 이유**: ED-01은 혼합 선택(일부 완료/일부 미완료) 가능하므로 경고가 적절, ED-02는 단일 항목이므로 비활성화가 명확

### 툴팁 센터링 — transform 미사용
- **결정**: `left: 50%` + 내부 `left: -50%` 패턴 사용
- **이유**: `tooltipFadeIn` 애니메이션의 `transform: translateY()`가 센터링용 `translateX(-50%)`를 덮어쓰는 버그 회피

### sourceUrl 필드 추가
- **결정**: `ProductDetail` 타입에 `sourceUrl` 추가, 소싱처별 더미 URL 생성
- **이유**: 상품 제목 클릭 시 소싱처 메인이 아닌 실제 상품 페이지로 이동해야 함
